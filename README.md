# Laboratory Activity #7: Order Management System

## Overview
This activity implements a simple Order Management System in Java for a small
online store. The program emphasizes abstraction, encapsulation, and safe
management of multiple orders using enums, interfaces, packages, and collections.

## Features
- Organize classes using logical packages
- Define fixed order states using an enum
- Abstract common order behavior using an abstract class
- Implement payment behavior using an interface
- Protect order data through encapsulation and validation
- Manage multiple orders using ArrayList
- Traverse and modify collections safely using Iterator

## Packages Used
- com.store.order
- com.store.payment
- com.store.main

## Classes and Interfaces Used
- Order (Abstract Class)
- OnlineOrder
- OrderStatus (Enum)
- Payable (Interface)
- OrderApp

## Methods Used
- processOrder()
- getOrderSummary()
- pay()
- Getter and setter methods
- main()

## OOP Concepts Applied
- Abstraction
- Encapsulation
- Interfaces
- Enums
- Package Organization
- Polymorphism
- Collections with Iterators
